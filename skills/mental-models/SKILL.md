## 1. Procrustean Bed / 普罗克鲁斯特之床

**核心问题**：用户是不是在**砍现实的腿**让它装进框架？

普罗克鲁斯特把不合床长度的客人砍腿或拉长。同理：
- 用高斯分布拟合厚尾世界
- 用 OKR 套创意工作
- 用 KPI 评估创业早期项目
- 用"用户调研"决定真正颠覆性产品

**何时调用**：用户在用一个**通用框架**（任何缩写词：SWOT、PEST、5W2H、OKR、SMART）评估一个**特殊**对象时。

**该问的**："这个框架在哪里把现实砍掉了一条腿？哪些重要维度它根本量不了？"

---

## 2. Hormesis / 毒物兴奋效应

**核心问题**：能不能给系统**适度施压**让它变强？

小剂量的毒变成解药。小剂量压力让系统更强：
- 肌肉需要被撕裂才能长大
- 免疫系统需要接触病原体（疫苗）
- 创业团队需要短期资金压力激发创造力
- 孩子需要少量挫折建立心理韧性

**反模式**：过度保护 = 制造脆弱（无菌儿童、零利率市场、过度补贴产业）。

**何时调用**：用户在描述一个"完全屏蔽风险"的设计时。问"你是不是在制造火鸡？"

**剂量原则**：剂量决定毒还是药。Hormesis 不是"压力越大越好"，是"小剂量压力 + 充分恢复"。

---

## 3. Minority Rule / 少数规则

**核心问题**：少数人的**不妥协**会不会决定全局？

一个**顽固的不可妥协少数**可以胁迫**灵活的多数**。例子：
- 全家有一个素食者 → 全家聚餐变素食
- 一个班级有一个花生过敏者 → 学校全面禁花生
- 一个市场有 5% 的清真消费者 → 几乎所有大众食品认证清真
- 一个 PR 灾难发生 → 整个公司决策被舆论挟持

**何时调用**：
- 用户在做大众市场决策时（多数偏好可能不重要，谁不可妥协才重要）
- 用户在估"民意"时（民意是均值，决策受最顽固那批人驱动）
- 用户在设计妥协方案时（不可妥协的一方根本不接受妥协，方案设计就废了）

## 4. Green Lumber Fallacy / 绿木材谬误

**核心问题**：**懂理论 ≠ 懂实战**——用户是不是在为"理论上对"的人/方案赋权，而非为"真在玩并赚钱"的？

塔勒布讲的故事：一个绿木材交易员赚大钱，但他以为"绿木材"是绿色油漆的木材——其实是没干透的新鲜木材。**理论错，实战对**。另一个交易员理论全对，破产了。

**何时调用**：
- 用户在引用专家/分析师/顾问预测时（他们有实战吗？还是纸上谈兵？）
- 用户自己在用理论否决直觉判断（实战者的直觉常优于体系化分析）
- 用户在评估候选人/合作方时（有 PhD ≠ 能落地）

**核查问题**：
- 这个人/方案有**真实暴露**吗？输了赔什么？
- 历史业绩 vs 理论自洽性——谁更重要？

# Pre-Mortem (Taleb 版) / 预演死亡

## 何时使用

- 用户即将做**重大决策**（创业、跳槽、大额投资、签长期合同）
- 用户**只在想成功怎样**，没认真想失败怎样
- 用户表现出**叙事谬误**苗头（"这肯定能成因为……"）

## 与 Kahneman 版 Pre-Mortem 的区别

| Kahneman 版 | Taleb 版 |
|---|---|
| 假设项目失败，列出可能原因 | 假设项目失败，列出**触发失败的暴露面** |
| 关注概率高的失败模式 | 关注**最坏的、低概率高破坏的**失败模式 |
| 目的：提高成功概率 | 目的：**确保失败不致命** |
| 接受失败可能性 | 拒绝**不可承受**的失败可能性 |

## 五步预演法

### 1. 时间穿越
"现在是 [3 年后]，你的 [决策] **彻底失败了**。不是小挫折，是**毁灭性失败**。"

### 2. 列出失败模式（强制三类）
- **可预见的常规失败**：竞争、执行、市场（这些通常被想到）
- **黑天鹅式毁灭**：监管、战争、瘟疫、技术颠覆（这些通常被忽略）
- **慢性脆弱积累**：不是某一刻爆掉，而是 18 个月慢慢腐蚀（最危险）

### 3. 致命性分级
对每个失败模式问：

| 等级 | 含义 | 容忍 |
|---|---|---|
| 可恢复 | 损失部分资金/时间，可以东山再起 | 可承担 |
| 重创 | 几年走不出阴影，但还活着 | 谨慎承担 |
| **不可逆毁灭** | 破产/坐牢/死亡/家庭破碎 | **绝不承担** |

### 4. 切肤之痛核对
- 失败时**谁承担**？只是用户吗？还是配偶/孩子/合伙人也被拖入？
- 用户**有没有自欺**：把"破产"重新命名为"重新开始"，把"坐牢风险"叫成"合规灰色地带"？

### 5. 重构方案
如果第 3 步出现**任何不可逆毁灭**，决策必须重构：
- 削减暴露规模（不是不做，是少做）
- 增加保险/对冲（不是消极，是凸性配置）
- 拒绝该决策（很多时候这才是答案）

## 输出格式

```
死亡时间： [3 年后 / 5 年后]
死法 1（常规）：   ……（致命性等级：可恢复 / 重创 / 毁灭）
死法 2（黑天鹅）： ……（致命性等级：……）
死法 3（慢性）：   ……（致命性等级：……）
切肤之痛：        ……（受牵连的人）
不可逆毁灭项：    ……（如有）
重构建议：        ……
```

## 反模式

- "我已经想过失败了，但我相信我能成" → 信念不是分析，列出来
- "失败了大不了重来" → 这是叙事，不是核算。重来的代价具体是什么？
- "失败概率很低" → 不要估概率，估**暴露面**

# Fragility Scan / 脆弱性扫描

## 何时使用

- 用户描述一个**系统**（公司、组合、关系、代码、计划），询问"是否稳"
- 用户做决策前希望**预先识别**最薄弱环节
- 用户**过度自信**，需要冷水

## 与传统风险评估的区别

| 传统风险评估 | 脆弱性扫描 |
|---|---|
| 估算"出事概率" | 定位"出事位置" |
| 量化期望损失 | 量化**最大可承受损失** vs **可能损失** |
| 关注已知威胁 | 主动找**未被想到的暴露面** |
| 输出风险评分 | 输出**先死的部位排序** |

## 七大脆弱性源（扫描清单）

### 1. 单点故障 / Single Point of Failure
- 这个系统有没有"一处失败全盘崩"的节点？
- 关键人？关键供应商？关键 API？关键账户？

### 2. 过度优化 / Optimization Trap
- 系统在某个指标上过度优化吗？
- 库存为零、缓冲为零、冗余为零通常 = 凹结构

### 3. 隐藏杠杆 / Hidden Leverage
- 名义杠杆 vs **真实杠杆**（包括或有负债、连带责任、担保）
- 心理杠杆（过度承诺、面子工程）也算

### 4. 路径依赖 / Path Dependency
- 系统是否依赖某条特定路径？路径中断时还能否运转？
- 远程团队中断网？团队解散后知识库丢失？

### 5. 模型脆弱性 / Model Fragility
- 决策依赖的预测模型在**输入分布外**会怎样？
- 高斯模型在厚尾世界 = 灾难
- 历史回测 = 在已知数据上拟合，未知冲击下失效

### 6. 委托代理 / Principal-Agent
- 决策者和承担后果的人是同一个吗？
- 不是 = 切肤之痛缺失 = 系统级脆弱

### 7. 沉默证据 / Silent Evidence
- 系统的"成功记录"是否包含幸存者偏差？
- 死掉的同类案例没被计入分析？

## 扫描流程

1. **让用户描述对象**：什么系统？什么目标？已经运转多久？
2. **逐项扫描七个源**：每项给出"低/中/高"风险，并写出具体证据
3. **不可承受损失定位**：哪个脆弱性触发会带来不可逆毁灭？
4. **优先级排序**：先修最不可承受的，不是最可能的
5. **强韧化建议**：每个高风险项给一个 Via Negativa 风格的减法建议（删/隔离/降低暴露）

## 输出格式

```
扫描对象：[简述]

脆弱性源扫描：
  1. 单点故障：     [低/中/高] - [具体证据]
  2. 过度优化：     [低/中/高] - [具体证据]
  3. 隐藏杠杆：     [低/中/高] - [具体证据]
  4. 路径依赖：     [低/中/高] - [具体证据]
  5. 模型脆弱性：   [低/中/高] - [具体证据]
  6. 委托代理：     [低/中/高] - [具体证据]
  7. 沉默证据：     [低/中/高] - [具体证据]

不可承受损失项：[排序后的 1-3 个]

强韧化建议（Via Negativa 优先）：
  - [删除什么 / 隔离什么 / 降低什么]
```

## 反模式

- "我已经做了风险管理" → 让用户具体说每一项，不要接受抽象
- "概率很低不用担心" → 重新引导到暴露面而不是概率
- "我们有保险" → 保险公司也会破产，对手风险呢？
